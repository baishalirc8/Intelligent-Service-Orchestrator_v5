HOLOCRON AI Technical Documentation

Comprehensive technical reference for the Automation Orchestration Platform

1. Platform Overview

HOLOCRON AI is an enterprise-grade SaaS Automation Orchestration Platform that transforms IT operations management through a human-first AI approach. The platform implements a CIO-level organizational hierarchy where departments are represented as "Crews" and roles as "Agents", inspired by CrewAI multi-agent orchestration patterns.

Core Value Proposition

  • Human-First Design — Assign human team members to IT roles first, then optionally enable AI shadow agents to augment capacity
  • Autonomous Infrastructure Management — Proactive monitoring, AI-driven discovery, and closed-loop remediation
  • ITIL-Aligned ITSM — Full incident, problem, change, and service request lifecycle management
  • Intelligent Cost Optimization — Template caching, cooldown guards, and dedup checks reduce AI API costs by up to 91%
  • Enterprise Orchestration — Committee-driven workflows, on-call management, and knowledge-driven task execution

6. AI Shadow Agents

AI Shadow Agents are the autonomous digital workers that augment human team members. When enabled on a role subscription, the AI agent inherits the role's responsibilities and AI capabilities, then operates within defined guardrails.

AI Agent Lifecycle

  1. Activation — User enables AI shadow on a role subscription
  2. Profile Generation — AI generates a best-practice metric profile for the role
  3. Asset Assignment — Discovered assets are mapped to the agent based on type/protocol
  4. Monitoring — Agent begins continuous monitoring of assigned assets
  5. Alerting — Agent generates alerts when thresholds are breached
  6. Remediation — For critical alerts, AI proposes remediation scripts for human approval

9. Discovery & Probes

The Discovery system enables automated network scanning and device identification through deployable probe agents.

Probe Architecture

Three coupling modes define how probes operate relative to the central server:

ModeConnectivityCapabilities
Coupled Constant server connection Real-time telemetry, immediate command execution
Semi-Autonomous Offline-capable with buffering Store-and-forward sync, local AI reasoning, JSONL buffer
Fully Autonomous Zero server dependency Container-based, on-device AI, self-remediation

Probe Agent Types

Kernel-Direct (Recommended)

Node.js probes reading directly from /proc and /sys kernel virtual filesystems. No shell dependencies. Run in distroless containers (~30MB).

  • probe-coupled.ts — Real-time telemetry
  • probe-semi.ts — Buffered with local AI
  • probe-auto.ts — Self-remediating autonomous

Shell Script (Legacy)

Bash/PowerShell probes requiring OS userspace tools (curl, top, free, df).

  • holocron-probe.sh — Linux coupled
  • holocron-probe-semi.sh — Linux semi-autonomous
  • holocron-probe-auto.sh — Linux autonomous
  • holocron-probe.ps1 — Windows coupled

Probe Clustering

Horizontal scaling for probes via cluster nodes. Each node contributes metric collection capacity based on hardware tier:

19. AI Cost Management

Three-layer cost optimization system protecting all 17+ OpenAI API call sites:

Layer 1: Template Cache

Caches AI-generated responses by asset type and agent role. At Enterprise scale (500 assets, ~20 types), reduces monthly cost from ~$2,373 to ~$204 (91% savings). 30-day TTL.

Layer 2: Cooldown Guards

Per-endpoint rate limits preventing redundant API calls. Validates preconditions before consuming the cooldown window.

EndpointCooldown
Alert Analysis / Remediation5 min
Calibration30 min
Discovery / Topology15 min
Health Refresh / Notifications5-10 min
Deep Scan10 min
Remediation Scripts2 min
Agent Chat5 sec

Layer 3: Deduplication Checks

Before generating remediation scripts, the system checks the database for existing tasks matching the same recommendation. Returns cached results when found.

25. Task Management

AI-powered autonomous task execution with Knowledge Base integration.

KB-First Execution

Before consuming AI tokens, the task executor searches the Knowledge Base for matching articles. If a strong keyword match is found (3+ overlapping words with 100+ char content), the KB article is returned directly — zero tokens consumed. Otherwise, relevant KB articles are injected as context for more accurate AI responses.

Task Lifecycle

Queued→ Executing (AI)→ Completed / Failed

Features

  • Run Agent — One-click AI task execution
  • Save to KB — Publish completed task outputs as Knowledge Base articles
  • Retry — Re-execute failed tasks
  • Priority Levels — Critical, High, Medium, Low

34. Security Architecture

Authentication Security

  • Password hashing with bcrypt (automatic salt)
  • Server-side sessions stored in PostgreSQL (not client-side tokens)
  • Session secrets via environment variables
  • CSRF protection through same-origin session cookies

Authorization & IDOR Protection

  • All :id routes verify resource ownership against the authenticated user
  • User-scoped queries prevent cross-tenant data access
  • requireAuth middleware on all protected endpoints

API Security

  • Request body validation using Zod schemas
  • Server-side ITIL status transition validation (rejects invalid state changes)
  • Rate limiting on AI endpoints via cooldown guards
  • Input sanitization on all user-provided content

AI Security

  • API keys stored as environment secrets (never exposed to frontend)
  • AI provider credentials encrypted at rest
  • Remediation scripts require human approval before execution
  • Autonomous probe self-remediation limited to safe operations (kill process, clear cache, cleanup disk, throttle workload)
